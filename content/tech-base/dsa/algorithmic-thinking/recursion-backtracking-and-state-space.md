# 递归、回溯与状态空间怎么理解

## 这个知识点在解决什么问题
很多人会写递归，却不知道为什么会超时、为什么会重复搜索。递归和回溯的本质，是在状态空间里系统地展开选择，再在必要时撤销选择。理解状态空间，才能知道什么时候该递归，什么时候该剪枝，什么时候该换动态规划。

## 递归在表达什么
递归不是“函数调用自己”这么简单，它在表达：
- 一个问题可以拆成规模更小的同类子问题
- 每一层都共享同一个问题结构

例如二叉树遍历、目录遍历、组合枚举，都天然适合递归表达。

## 回溯在表达什么
回溯比普通递归多了一步：撤销选择。

```text
做一个选择
  -> 进入下一层
  -> 返回后撤销这个选择
```

这使它特别适合：
- 排列组合
- 子集枚举
- 路径搜索
- 约束条件筛选

## 什么叫状态空间
状态空间就是“所有可能中间状态构成的搜索树”。

例如长度为 3 的二进制选择：

```text
        []
      /    \
    [0]    [1]
   /  \    /  \
[00][01][10][11]
```

如果你不理解自己在搜索什么状态，就很难写对递归终止条件和剪枝条件。

## 剪枝为什么重要
回溯最怕无意义搜索。剪枝的意思是：
- 某条路径已经不可能成为合法答案
- 那就不必继续往下展开

例如组合求和时，当前和已经超过目标值，就可以提前返回。

## 和动态规划的边界
如果同一个子问题被重复计算很多次，且结果可以复用，那么可能已经更适合动态规划或记忆化搜索。

## 常见误区
- 把递归当写法技巧，不先定义状态和终止条件
- 回溯时忘记撤销选择
- 不剪枝，导致搜索空间爆炸
- 能用循环直接解决的问题也硬套递归
