# 单调栈、单调队列与滑动窗口

## 这个知识点在解决什么问题
很多题目表面上是在“遍历数组”，本质上是在维护一个局部最优结构。单调栈、单调队列和滑动窗口，解决的是“如何在顺序扫描时，持续保留当前最有价值的一小部分信息”，从而避免每次都回头重算。

## 滑动窗口在做什么
滑动窗口适合处理一段连续区间：
- 固定长度窗口
- 满足某个条件的最短/最长区间
- 连续子串、连续子数组问题

核心动作是：
- 右边扩张
- 左边收缩
- 维护窗口内部统计量

```text
[ left ........ right ]
```

## 单调栈在做什么
单调栈通常解决“离当前元素最近、且比它大/小的元素是谁”。

典型题型：
- 下一个更大元素
- 每日温度
- 柱状图最大矩形

它维护的是一个单调递增或递减的栈，保证一旦新元素到来，就能快速淘汰不再有价值的旧元素。

## 单调队列在做什么
单调队列通常和窗口一起出现，核心是：
- 窗口在移动
- 你想持续知道窗口内最大值或最小值

普通队列只能保顺序，单调队列还能保“当前候选最大/最小值”的结构。

## 一个典型例子
### 长度为 `k` 的窗口最大值
如果每次窗口移动都重新扫描 `k` 个元素，复杂度会很高。单调队列的思路是：
- 新元素入队前，先把队尾比它更差的元素弹掉
- 队首永远是当前窗口最优值
- 窗口左边离开时，如果正好是队首，再弹出

这样每个元素最多进出一次，整体通常是线性的。

## 为什么这和“测试开发”也有关
很多测试开发场景需要处理连续日志、时间窗口指标、最近 N 次状态变化：
- 最近 5 分钟错误率
- 连续帧卡顿检测
- 一段时间内接口延迟最大值

这些问题背后的数据处理思路，就和窗口、单调结构很接近。

## 常见误区
- 看到窗口题就直接双重循环
- 把单调结构当模板背，却不知道它维护的“单调性”代表什么
- 忘了先判断元素是否已离开窗口
