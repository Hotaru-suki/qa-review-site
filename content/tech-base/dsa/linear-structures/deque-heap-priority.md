# 双端队列、堆与优先级结构

## 摘要

当普通数组、栈、队列不够表达问题时，双端队列和堆会提供更贴近问题本身的访问方式。双端队列强调“头尾都能高效进出”，堆强调“始终快速拿到当前最优元素”。它们都不是为了更复杂，而是为了让特定操作更便宜。

---

## 核心问题

- 双端队列为什么适合头尾都频繁操作的场景
- 堆到底是怎么组织的，为什么能快速取最值
- 优先队列和“每次都重新排序”有什么本质区别

---

## 1. 双端队列是什么

双端队列，`deque`，可以理解成“头和尾都能高效插入删除的队列”。

```text
front -> [10] [20] [30] <- back
```

它支持：

- 从头插入
- 从头删除
- 从尾插入
- 从尾删除

这和普通队列只允许“尾入头出”不一样。

### 为什么它适合滑动窗口

滑动窗口问题里，元素会不断：

- 从尾部加入新元素
- 从头部移除过期元素

如果还要求窗口内维护单调性，就会同时在头尾做调整，`deque` 就非常自然。

---

## 2. 堆的底层结构长什么样

堆通常是一棵“近似完全二叉树”，但常用数组来存。

例如一个小根堆可以这样表示：

```text
        2
      /   \
     5     7
    / \   /
   9   6 8
```

放到数组里通常是：

```text
index:  0  1  2  3  4  5
value: [2, 5, 7, 9, 6, 8]
```

对于下标 `i`：

- 左孩子通常是 `2*i + 1`
- 右孩子通常是 `2*i + 2`
- 父节点通常是 `(i - 1) // 2`

这就是为什么堆可以不用链式节点，也能维护树结构。

---

## 3. 堆为什么能快速拿到当前最优

以小根堆为例，它维护的不是“全局有序”，而是“父节点不大于孩子节点”。

所以：

- 最小值永远在堆顶
- 但堆里其他元素不一定完全排好序

这意味着：

- 取最小值是 `O(1)`，直接看堆顶
- 插入和删除堆顶时，只需沿树高调整，复杂度通常是 `O(log n)`

堆并不保证整体有序，它只保证“最该被取出的那个元素在顶部”。

---

## 4. 插入和删除的过程是什么

### 插入

插入新元素时，通常先放到数组末尾，再向上调整。

例如往小根堆插入 `3`：

```text
原堆数组: [2, 5, 7, 9, 6, 8]
追加后:   [2, 5, 7, 9, 6, 8, 3]
```

然后做“上浮”：

- 比较 `3` 和父节点 `7`
- `3 < 7`，交换
- 再和新的父节点比较

最终变成：

```text
[2, 5, 3, 9, 6, 8, 7]
```

### 删除堆顶

删除最小值时：

1. 取走堆顶
2. 用最后一个元素补到堆顶
3. 再向下调整

例如：

```text
原堆: [2, 5, 3, 9, 6, 8, 7]
删顶: 取走 2，用 7 补上
暂时: [7, 5, 3, 9, 6, 8]
```

然后做“下沉”：

- `7` 和更小的孩子 `3` 交换
- 继续往下比较

---

## 5. 时间复杂度为什么是这样

| 操作 | 时间复杂度 | 原因 |
| --- | --- | --- |
| 查看堆顶 | `O(1)` | 直接取数组首元素 |
| 插入 | `O(log n)` | 最多沿树高上浮 |
| 删除堆顶 | `O(log n)` | 最多沿树高下沉 |
| 建堆 | `O(n)` | 从下往上调整 |
| 空间 | `O(n)` | 存储全部元素 |

为什么不是 `O(1)`？

因为虽然堆顶可直接拿，但插入和删除后要重新恢复堆性质，而恢复最多要走树高，大约是 `log n` 层。

---

## 6. 优先队列和排序的区别

很多人会把堆理解成“另一种排序结构”，这不准确。

### 排序关注的是

- 所有元素最终整体有序

### 优先队列关注的是

- 每次都能快速拿到当前优先级最高的元素
- 同时还允许不断加入新元素

例如任务调度：

- 任务会不断进入系统
- 每次只关心当前最高优先级任务是谁

这时如果每来一个任务都全量排序，代价太高；用堆维护优先队列更自然。

## 7. 一个简化实现示例

下面用 Python 写一个小根堆的核心操作：

```python
class MinHeap:
    def __init__(self):
        self.data = []

    def push(self, x):
        self.data.append(x)
        i = len(self.data) - 1
        while i > 0:
            p = (i - 1) // 2
            if self.data[p] <= self.data[i]:
                break
            self.data[p], self.data[i] = self.data[i], self.data[p]
            i = p

    def pop(self):
        if not self.data:
            return None
        top = self.data[0]
        last = self.data.pop()
        if self.data:
            self.data[0] = last
            self._sift_down(0)
        return top

    def _sift_down(self, i):
        n = len(self.data)
        while True:
            left = 2 * i + 1
            right = 2 * i + 2
            smallest = i
            if left < n and self.data[left] < self.data[smallest]:
                smallest = left
            if right < n and self.data[right] < self.data[smallest]:
                smallest = right
            if smallest == i:
                break
            self.data[i], self.data[smallest] = self.data[smallest], self.data[i]
            i = smallest
```

这段代码最重要的不是背下来，而是看懂：

- 插入靠上浮恢复堆性质
- 删除堆顶靠下沉恢复堆性质

## 8. 一个工程或算法场景

例如定时任务系统中，总是要先执行“最早到期”的任务：

- 任务持续加入
- 每次只取当前最早任务
- 处理完后再取下一个

这时用小根堆维护“到期时间最小”的任务就很自然。

又比如 Top K 问题：

- 数据不断流入
- 只想保留最大的 K 个

可以维护一个大小为 K 的小根堆，让堆顶始终是当前第 K 大。

---

## 小结

双端队列和堆都不是为了“更复杂”，而是为了让特定访问模式更高效。双端队列优化头尾进出，堆优化动态最优元素维护。只要你能看清问题到底是在做“窗口移动”还是“持续取最优”，结构选型就会自然很多。
