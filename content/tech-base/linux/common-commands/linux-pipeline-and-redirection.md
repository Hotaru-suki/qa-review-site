# 管道、重定向与命令组合

## 这个知识点在解决什么问题
Linux 命令真正强的地方，不是单个命令会多少，而是能不能把多个命令像积木一样组合起来。管道和重定向解决的就是：一个命令的输出，如何直接成为下一个命令的输入；结果是显示到屏幕、写到文件，还是追加到日志。

## 重定向在做什么
- `>`：覆盖写入文件
- `>>`：追加写入文件
- `<`：把文件作为输入

例如：

```bash
grep ERROR app.log > error.log
```

意思是把筛出来的错误日志写进 `error.log`。

## 管道在做什么
管道 `|` 的核心是把前一个命令的标准输出，直接喂给后一个命令。

```bash
cat app.log | grep ERROR | wc -l
```

这不是三个孤立命令，而是一条数据流：

```text
app.log -> grep ERROR -> 计数
```

## 为什么这对排障特别重要
排障时很少有“一个命令直接解决”的情况，更常见的是：
- 先定位文件
- 再筛关键字
- 再看上下文
- 再统计频次

例如：

```bash
grep 'orderId=123' app.log | tail -n 20
```

它就很适合快速查看某笔业务最近相关日志。

## 测试开发常见用法
- 从大日志里筛某个用户、订单或 traceId
- 查看接口错误码分布
- 统计异常请求数量
- 把一次排查结果暂存到文件里，后续继续分析

## 常见误区
- 只会单个命令，不会把它们串起来
- 不理解覆盖和追加的区别，误把旧日志清空
- 一上来就 `cat 大文件 | grep`，但其实可以直接 `grep` 文件
