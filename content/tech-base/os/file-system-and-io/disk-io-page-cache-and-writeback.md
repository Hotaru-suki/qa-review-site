# 磁盘 IO、页缓存与回写机制

## 这个知识点在解决什么问题
很多人以为“程序写文件”就等于“数据立刻写进磁盘”。实际上，文件 IO 往往先经过页缓存，再由内核择机回写到磁盘。理解这条路径，才能明白为什么：
- 写入调用返回很快，但磁盘并没有马上落盘
- 磁盘忙时系统会出现抖动
- 某些场景必须显式刷盘

## 页缓存是什么
页缓存是内核为了提升 IO 性能，在内存里缓存文件页的数据结构。

读取时：
- 热数据可能直接从缓存返回，不必每次都打磁盘

写入时：
- 进程先把数据写到缓存页
- 系统稍后再统一回写到磁盘

## 回写在做什么
回写是把脏页真正落到磁盘的过程。它的好处是：
- 合并小写入
- 提升吞吐

代价是：
- 返回成功不一定代表已经物理落盘
- 突发回写可能带来抖动

## 为什么排障时要关心它
你看到“写接口变慢”时，可能并不是业务逻辑慢，而是：
- 页缓存积累太多脏页
- 回写压力大
- 磁盘 IO 被打满

## 典型场景
### 日志大量刷盘
- 平时写日志很快
- 压测时突然出现 IO 抖动

### 数据持久化
- 应用以为数据已经安全写入
- 实际还在页缓存里

所以数据库、中间件或关键持久化逻辑会更谨慎地处理刷盘策略。

## 常见误区
- 把一次 `write` 返回成功理解成“数据已经上盘”
- 只看应用耗时，不看磁盘队列、回写和脏页情况
- 认为磁盘问题一定表现为接口完全不可用，其实很多时候先表现为长尾抖动
