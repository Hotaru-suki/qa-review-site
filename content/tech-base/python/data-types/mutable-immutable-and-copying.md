# 可变、不可变与拷贝语义

## 这个知识点在解决什么问题
Python 初学者最常见的坑，不是语法不会，而是不知道一个对象是原地被改了，还是新建了副本。可变、不可变与拷贝语义解决的是：为什么有些操作会影响原对象，有些不会；为什么函数传参后外面有时会变，有时不会。

## 什么是可变和不可变
- 不可变对象：创建后内容不能原地改，例如 `int`、`str`、`tuple`
- 可变对象：可以原地修改，例如 `list`、`dict`、`set`

## 为什么这会影响代码行为
```python
a = [1, 2]
b = a
b.append(3)
```

这里 `a` 也会变，因为 `a` 和 `b` 指向同一个列表对象。

而：

```python
x = "ab"
y = x
y += "c"
```

`x` 不会变，因为字符串不可变，`y += "c"` 会创建新对象。

## 浅拷贝和深拷贝
- 浅拷贝：复制外层容器，内层对象仍然共享
- 深拷贝：递归复制内外层对象

例如：

```python
import copy

data = {"a": [1, 2]}
shallow = copy.copy(data)
deep = copy.deepcopy(data)

shallow["a"].append(3)
```

这时：
- `data["a"]` 会变，因为浅拷贝只复制了最外层字典
- `deep["a"]` 不会变，因为深拷贝把内层列表也复制了

## 函数传参为什么经常让人误会
Python 传的是对象引用。

如果函数里原地修改可变对象：

```python
def add_item(items):
    items.append("x")
```

外部列表会变。

如果是重新绑定变量：

```python
def reset(items):
    items = []
```

外部原对象不会变，因为这里只是让局部变量重新指向了新列表。

## 为什么测试开发要懂它
- 构造测试数据时，多个用例共用同一模板对象
- 一个用例修改了嵌套字段，别的用例也被污染
- 缓存、配置、默认参数里对象共享导致偶发问题

## 一个典型坑
### 用模板字典生成测试数据

```python
template = {"headers": {"token": "abc"}, "payload": {"id": 1}}
case1 = template.copy()
case2 = template.copy()

case1["headers"]["token"] = "def"
```

很多人以为 `case2` 不受影响，但实际上：
- 外层字典复制了
- 内层 `headers` 还是同一个对象

这类问题在接口自动化里非常常见。

## 常见误区
- 以为赋值就是复制
- 浅拷贝后就以为完全独立
- 在函数默认参数里放可变对象
- 看到对象“值一样”就误以为“不是同一个对象”
