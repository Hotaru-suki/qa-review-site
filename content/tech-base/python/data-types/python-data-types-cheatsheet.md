# Python 数据类型常用操作

## 摘要

Python 数据类型的常用操作，不该被背成一张方法表，而应该被理解成“不同数据语义下，最自然的读、改、增、删方式”。真正掌握这些操作，关键不在 API 数量，而在于你是否理解可变性、访问方式、时间复杂度倾向以及这些操作到底是在原地修改，还是创建新对象。

---

## 核心问题

- 为什么同样是“存一组值”，`list`、`tuple`、`set`、`dict` 的操作方式完全不同
- 可变对象和不可变对象会怎样影响赋值、传参和修改行为
- 常用操作到底是在“原地修改对象”还是“生成新对象”

---

## 1. 先建立总认知：操作由类型语义决定

Python 的常用操作之所以看起来很多，是因为不同类型本来就在表达不同的数据语义：

- `str` 表示文本序列
- `list` 表示有序、可变、可重复的集合
- `tuple` 表示有序但不可变的结构
- `set` 表示无序、去重的集合
- `dict` 表示键到值的映射

所以学习操作时不要先背方法名，而要先问：

- 这个类型被设计出来，是为了解决什么数据组织问题

---

## 2. 数值和布尔值：重点在表达式语义

`int`、`float`、`bool` 的高频点不在方法，而在表达式计算和条件判断。

### `int`

更适合：

- 计数
- 下标
- 离散状态

### `float`

更适合：

- 近似数值计算

但要特别注意：

- 精度问题

### `bool`

本质上是条件结果，常和比较、逻辑表达式一起出现。

最重要的是分清：

- `/` 是真除法
- `//` 是向下取整除法
- `%` 常用于取余和周期判断
- `and`、`or`、`not` 参与的是逻辑求值

---

## 3. 字符串操作为什么必须先记“不可变”

`str` 最关键的性质是：

- 不可变

这意味着：

- 切片、替换、拼接通常会产生新字符串
- 不能通过下标原地改字符
- 多次拼接大字符串时要注意开销

例如：

```python
s = "hello"
t = s.replace("h", "y")
print(s)  # hello
print(t)  # yello
```

这里 `replace` 返回的是新对象，并没有改原来的 `s`。

常见高频操作包括：

- 切片：取子串
- `split`：按分隔规则拆成列表
- `join`：把一组字符串拼回文本
- `replace`：替换内容
- `strip`：去掉首尾噪声

如果先理解“字符串不可变”，这些操作行为就会很自然。

---

## 4. 列表操作为什么最容易出副作用

`list` 是 Python 里最常用的数据类型之一，因为它：

- 有序
- 可变
- 可重复

高频操作可以按目的理解：

- `append`、`extend`、`insert`：加元素
- `pop`、`remove`、`del`：删元素
- 切片：取子序列
- `sort`、`reverse`：原地调整顺序
- 列表推导式：从已有数据生成新列表

最容易出错的点不是方法名，而是：

- 哪些操作原地修改
- 哪些操作返回新结果

例如：

```python
nums = [3, 1, 2]
nums.sort()
print(nums)  # [1, 2, 3]
```

`sort()` 是原地修改。

而：

```python
nums = [3, 1, 2]
new_nums = sorted(nums)
print(nums)      # [3, 1, 2]
print(new_nums)  # [1, 2, 3]
```

`sorted()` 返回新列表，不改原对象。

这个差别在脚本和面试里都非常高频。

---

## 5. 元组为什么不只是“不能改的 list”

很多初学者会把 `tuple` 理解成：

- 不能改的 `list`

这太窄了。

`tuple` 更适合表达：

- 一组位置含义固定的数据
- 不希望被误修改的结构
- 可作为字典 key 或集合元素的可哈希对象

例如：

- `(x, y)`
- `(name, age)`
- 数据库一行结果

这类数据重点不在“我要不要改”，而在：

- 结构本身就是稳定的

---

## 6. 集合操作的核心是“去重”和“成员判断”

`set` 最大的价值不是存一批数据，而是：

- 自动去重
- 快速判断某元素是否存在
- 做交集、并集、差集

典型场景包括：

- 标签去重
- 黑名单、白名单判断
- 两批数据的重合和差异分析

例如：

```python
a = {1, 2, 3}
b = {2, 3, 4}
print(a & b)  # {2, 3}
print(a | b)  # {1, 2, 3, 4}
print(a - b)  # {1}
```

如果场景需要顺序，`set` 就不是合适选择；如果场景主要关心“有没有”和“重复不重复”，`set` 会非常自然。

---

## 7. 字典操作为什么是工程里的核心能力

`dict` 是 Python 项目中最常见的数据组织方式之一，因为现实里的很多数据本来就是：

- 字段名 -> 字段值

学习字典操作要重点理解：

- 通过 key 访问比按位置访问更有语义
- key 必须是可哈希对象
- 字典天然适合表示 JSON、配置、对象属性和统计结果

常见高频操作如：

- 读取：`d[key]`、`get`
- 新增或更新：`d[key] = value`、`update`
- 遍历：`items()`、`keys()`、`values()`
- 删除：`pop`、`del`

其中 `get` 和直接下标访问的差别非常重要：

```python
d = {"name": "alice"}
print(d.get("age"))      # None
print(d.get("age", 0))   # 0
# d["age"] 会抛 KeyError
```

这会直接影响你是希望：

- 缺失时给默认值
- 还是让程序明确失败

---

## 8. 可变性为什么会影响赋值和传参

Python 里很多“我只改了一处，怎么别的地方也变了”的问题，本质都和对象可变性有关。

例如：

```python
a = [1, 2]
b = a
b.append(3)
print(a)  # [1, 2, 3]
```

这里不是复制了一份列表，而是：

- `a` 和 `b` 指向同一个对象

而对于不可变对象：

```python
s = "abc"
t = s
t = t + "d"
print(s)  # abc
print(t)  # abcd
```

这里是新字符串被创建并绑定给了 `t`。

所以学习常用操作时，必须把：

- 值
- 对象
- 引用绑定

区分开，否则在函数传参、默认参数、嵌套容器里非常容易踩坑。

---

## 9. 一个更实用的学习顺序

如果要真正掌握数据类型操作，建议按下面顺序理解：

1. 这个类型表达什么语义
2. 它是可变还是不可变
3. 高频操作是在读、改、删，还是生成新对象
4. 哪些场景最适合用它

这样再去看方法名时，就不会觉得是零散 API。

---

## 10. 常见误区

### 10.1 把方法名当重点

真正重点是数据语义和对象行为。

### 10.2 分不清原地修改和返回新对象

这会直接带来副作用 bug。

### 10.3 不理解可变性

赋值、传参、默认参数都会变得很容易出错。

### 10.4 用错数据类型

很多问题不是“不会操作”，而是一开始就选错了结构。

---

## 小结

Python 数据类型的常用操作不适合死背成速查表，真正应该掌握的是：每种类型表达什么语义、它是可变还是不可变、常见操作是在修改原对象还是生成新对象，以及在什么场景下选它最自然。只要这些点清楚，方法名就会变成顺理成章的工具，而不是记忆负担。
