# 条件、循环与推导式

## 摘要

Python 的条件、循环和推导式，不只是语法糖，而是控制程序执行路径和批量处理数据的基础方式。真正要掌握的不是写法本身，而是何时该分支、何时该遍历、何时该提前退出、何时该把循环收缩成表达式。很多脚本 bug 都不在语法，而在条件漏写、循环边界错写、状态更新混乱或把复杂逻辑硬塞进推导式。

---

## 核心问题

- 条件语句和循环分别在控制什么
- `if`、`for`、`while` 在实际场景里该怎么选
- 为什么推导式常常更简洁，但并不适合所有逻辑
- 循环里的状态更新和退出条件为什么是 bug 高发点

---

## 1. 条件语句到底在控制什么

条件语句的本质，是根据当前状态选择不同执行路径。

例如：

```python
score = 85

if score >= 90:
    level = "A"
elif score >= 60:
    level = "B"
else:
    level = "C"
```

这里真正重要的不是会写 `if/elif/else`，而是你是否想清楚：

- 哪些条件互斥
- 哪些条件有优先级
- 有没有遗漏分支

很多 bug 并不是语法错，而是分支覆盖不完整。

---

## 2. `for` 和 `while` 分别适合什么

### `for`

更适合“遍历一组已知可迭代对象”。

例如：

```python
for user in users:
    print(user)
```

它强调的是：对集合里的每个元素做同样逻辑。

### `while`

更适合“只要条件还成立，就持续做某件事”。

例如：

```python
count = 0
while count < 3:
    print(count)
    count += 1
```

它强调的是：循环次数不一定一开始就固定，而是由状态变化控制。

所以最直观的区分是：

- 已知要遍历谁，优先 `for`
- 依赖条件持续判断，优先 `while`

---

## 3. 循环里最容易出问题的地方是什么

循环本身并不复杂，最容易出问题的是：

- 退出条件不完整
- 循环变量更新错了
- 在循环里修改外部可变状态
- 一边遍历，一边改同一个容器

例如：

```python
nums = [1, 2, 3, 4]
for x in nums:
    if x % 2 == 0:
        nums.remove(x)
```

这类代码经常产生意料之外的结果，因为你在遍历列表的同时又改了列表本身。

更稳的方式通常是构造新列表：

```python
nums = [x for x in nums if x % 2 != 0]
```

---

## 4. 推导式到底适合什么

推导式本质上是在表达：

- 遍历
- 过滤
- 生成新结果

最典型写法：

```python
users = ["alice", "bob", "carol"]
active_users = [user for user in users if user != "bob"]
```

推导式适合：

- 规则明确
- 表达式短
- 没有复杂副作用
- 目的就是生成一个新容器

不适合：

- 逻辑分支很多
- 要打日志
- 要捕获异常
- 要更新多个状态

推导式的目标是让简单数据变换更紧凑，不是把复杂过程硬压成一行。

## 5. 一个具体脚本场景

假设你要处理一组接口返回结果，筛出所有失败项并提取其 ID。

### 普通写法

```python
failed_ids = []
for item in results:
    if item["status"] != "ok":
        failed_ids.append(item["id"])
```

### 推导式写法

```python
failed_ids = [item["id"] for item in results if item["status"] != "ok"]
```

如果逻辑只是筛选和提取，推导式更直接。

但如果你还要：

- 记录失败原因
- 对异常结构做兼容
- 统计不同状态数量

就不该强行写成一条推导式。

## 6. `break`、`continue` 和 `else` 怎么理解

### `break`

表示提前结束整个循环。

```python
for x in nums:
    if x == target:
        print("found")
        break
```

### `continue`

表示跳过当前这一轮，直接进入下一轮。

```python
for x in nums:
    if x < 0:
        continue
    print(x)
```

### 循环 `else`

Python 的循环 `else` 表示：只有循环没有被 `break` 打断时才执行。

```python
for x in nums:
    if x == target:
        print("found")
        break
else:
    print("not found")
```

这个特性很多人不常用，但在“查找是否存在”类逻辑里很自然。

---

## 7. 一个边界条件错误示例

`while` 循环很容易因为状态没更新而写成死循环：

```python
count = 0
while count < 5:
    print(count)
```

这里缺少 `count += 1`，循环永远不会结束。

所以写 `while` 时一定要同时问自己：

- 条件什么时候会变
- 谁负责把状态推进到退出条件

这比记语法更重要。

---

## 8. 测试和脚本里怎么用得更稳

在自动化和数据处理脚本里，通常可以遵循几个简单原则：

- 纯遍历优先 `for`
- 条件驱动重复优先 `while`
- 简单过滤和映射优先推导式
- 一旦逻辑开始复杂，就回到普通循环
- 避免在循环里隐式改外部共享状态

这样代码会更容易读，也更容易排查。

---

## 小结

流程控制的核心，不是会写 `if`、`for` 和推导式，而是知道逻辑为什么在这里分叉、状态为什么在这里推进、循环为什么会结束，以及代码是否仍然保持清晰。条件和循环写得稳，本质上靠的是边界感和状态感，不是语法记忆。
