# 数据库与 SQL

## 摘要

数据库与 SQL 专题不是一堆零散语法，而是一整套“如何查数据、为什么会快、为什么会慢、并发时为什么会乱”的知识体系。这个概述页只负责给出专题地图，让你先明确这一专题包含哪些知识块，以及它们各自解决什么问题。

---

## 本专题结构

### SQL 基础与查询

- SQL 基础语法
- 连接与子查询
- 窗口函数与高频题
- 测试场景 SQL

这一部分解决“怎么查”。

### 数据库原理

- 数据库基础概念
- 事务与并发控制

这一部分解决“为什么这样存、这样约束、这样并发”。

### 性能优化与排查

- 索引与执行计划
- 慢查询与排查基础

这一部分解决“为什么慢、怎么定位瓶颈”。

### 笔面试高频题

- 数据库与 SQL 高频题

这一部分解决“如何把前面的知识讲清楚”。

---

## 阅读路径

1. 先学 SQL 基础语法与数据库基础概念
2. 再学连接、子查询、窗口函数和测试场景 SQL
3. 然后补事务、并发控制、索引与执行计划
4. 最后用慢查询和高频题把整体串起来

---

## 本专题重点

- SQL 负责“怎么查”，数据库原理负责“为什么会这样”
- 性能问题最终都要回到索引、执行计划、数据量和并发去理解
- 真正掌握这一专题，不只是会写 SQL，还要会解释、会排查、会判断取舍
