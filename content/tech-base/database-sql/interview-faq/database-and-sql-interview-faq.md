# 数据库与 SQL 高频问题串联

## 摘要

这篇内容不是单纯的问答清单，而是把数据库与 SQL 中最容易被反复追问的问题按主题串起来。它的作用不是让你死记答案，而是帮助你看清：哪些概念最容易混、哪些问题最常被组合追问，以及这些问题背后真正考察的到底是什么理解。

---

## 核心问题

- 数据库与 SQL 里最容易被反复追问的问题集中在哪些主题
- 为什么很多问题表面不同，底层其实考的是同一套理解
- 怎样把 SQL、索引、事务和排查思路串成统一知识图

---

## 1. 高频问题为什么总围着那几类转

面试里数据库问题虽然看起来很多，但高频点通常集中在：

- SQL 查询基础
- 连接、子查询和结果组织
- 索引与执行计划
- 事务与并发控制
- 慢查询和优化思路

这是因为数据库面试真正想看的是：

- 你会不会写 SQL
- 你理不理解 SQL 背后的执行代价
- 你能不能把查询、索引、事务和性能问题连起来讲

所以它并不是“零散题库”，而是一组会互相追问的知识链。

---

## 2. SQL 基础类问题在考什么

常见问题包括：

- `WHERE` 和 `HAVING` 的区别
- `COUNT(*)` 和 `COUNT(字段)` 的区别
- `LIKE`、`UNION`、`JOIN` 分别在解决什么问题

这类题表面上像语法题，底层其实在考：

- 你是否理解执行时机
- 你是否知道不同写法会影响结果集
- 你能不能把语法放回真实查询场景

也就是说，它们不是“会不会背定义”，而是“知道 SQL 在做什么”。

---

## 3. 连接、子查询和结果组织类问题在考什么

常见问题包括：

- `INNER JOIN` 与 `LEFT JOIN` 的区别
- `LEFT JOIN` 里 `ON` 和 `WHERE` 为什么会改变结果集
- 子查询和连接在思路上如何取舍

这类题真正想看的是：

- 你是否理解结果集怎么被构造出来
- 你是否知道一个条件放在不同位置会影响最终保留哪些行
- 你能不能解释结果变化，而不是只给结论

所以这类题是“结果组织逻辑题”，不是死记术语题。

---

## 4. 索引与执行计划类问题在考什么

常见问题包括：

- 索引为什么能提速
- 索引为什么有时会失效
- 联合索引和最左前缀到底限制了什么
- `EXPLAIN` 的重点字段应该怎么解读

这类题底层在考的是：

- 你是否理解查询为什么快或慢
- 你是否知道数据库不是“建了索引就一定走”
- 你能不能把 SQL 写法、数据分布和执行计划联系起来

这也是为什么数据库题里，“会不会看执行计划”很重要。

---

## 5. 事务与并发控制类问题在考什么

常见问题包括：

- ACID 分别在约束什么
- 脏读、不可重复读、幻读的本质区别
- 乐观锁和悲观锁分别在对抗什么风险

这类题看起来像原理题，实际上在考：

- 你是否理解多事务同时操作数据时为什么会乱
- 你是否知道不同隔离级别和锁策略是在做什么权衡
- 你能不能把并发问题落到真实业务场景

如果只背定义而不会举场景，这类题通常很难答稳。

---

## 6. 慢查询和优化类问题在考什么

常见问题包括：

- 深分页为什么慢
- `SELECT *` 为什么不推荐
- 为何建了索引仍然慢
- 报表或统计接口为什么在大数据量下超时

这类题的底层，其实是在看你有没有“执行代价意识”：

- 是执行慢，还是等待慢
- 是 SQL 写法问题，还是索引和数据量问题
- 是单条 SQL 慢，还是并发、锁和资源竞争导致慢

这类思维一旦建立，很多数据库面试题就不再是死记硬背。

---

## 7. 怎样把这些问题串成一张图

更合理的理解方式通常是：

```text
SQL 写法
  ↓
结果集和执行方式
  ↓
是否命中索引
  ↓
在并发和事务下是否稳定
  ↓
最终性能和排查路径
```

也就是说：

- SQL 决定你想查什么
- 索引决定怎么更快查
- 事务决定并发下查和改是否安全
- 排查决定慢了或错了时怎么定位

只要这条主线清楚，很多题看起来不同，实际上都能串起来。

---

## 8. 面试时怎么用这篇内容

更合理的用法通常是：

1. 先在对应专题文章里把概念和机制学透
2. 再回到这里检查哪些问题最容易混
3. 再用这篇内容补齐“概念之间如何连接”的视角

这样会比单独记问答结论更稳。

---

## 9. 常见误区

- 只记定义，不理解场景
- SQL、索引、事务分别会一点，但讲不成体系
- 遇到问题只会给结论，讲不清为什么

数据库面试真正拉开差距的，通常不是“知道多少词”，而是能不能把这些词讲成一套系统。

---

## 小结

数据库与 SQL 的高频问题并不是独立题库，而是一张知识网络中的高频入口。把这些问题背后的原理、边界、场景和执行代价串起来，专题理解会比单独背答案扎实得多。
