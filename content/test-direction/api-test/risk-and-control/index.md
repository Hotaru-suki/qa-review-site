# 异常、权限与风险控制

## 摘要

这一组知识点关注接口在异常输入、权限校验和风险场景下是否仍然可控，重点是把“错误请求”和“高风险请求”都挡住并表达清楚。它处理的是接口在边界条件和攻击面上的质量问题。

---

## 本组内容

- `健壮性测试`
  看错误输入和边界条件是否被正确拦截。
- `敏感数据、脱敏与审计线索`
  看接口有没有把不该暴露的信息带出去，以及出事后能否追。
- `兼容性测试`
  看不同版本或不同调用方式下接口是否稳定。
- `接口安全性测试`
  看敏感数据、越权和高风险调用是否被控制。
- `鉴权与 Token 机制`
  看身份和权限边界是否可靠。
- `错误码、状态码与异常处理`
  看失败是否被表达清楚。
- `批量操作、补偿与高风险写接口`
  看一次调用影响很多对象时如何防止写乱和半成功。

---

## 阅读重点

- 错误输入是否被正确拦截
- 身份和权限边界是否被真正管住
- 风险场景下系统是否会暴露敏感信息或写乱状态
