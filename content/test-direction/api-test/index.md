# 接口测试

## 摘要

接口测试专题只讲接口测试本身，不展开 UI 测试。这个专题的作用是把接口测试拆成一组可以逐个吃透的知识点，让你先知道接口测试到底由哪些部分组成、重点应该先抓什么。

---

## 本专题结构

### 基础认知

- 接口测试概述
- 接口测试流程
- 接口测试用例设计思路

### 核心质量维度

- 可用性测试
- 正确性测试
- 健壮性测试
- 稳定性测试
- 接口性能测试
- 数据一致性测试
- 幂等性测试
- 接口安全性测试
- 兼容性测试

### 工具与实践

- Postman / Apifox 调试
- pytest + requests 接口自动化
- JMeter 接口性能测试
- MySQL / Redis 数据校验
- Mock 在接口测试中的应用

### 进阶专题

- 接口测试常见面试题
- 鉴权与 Token 机制
- 错误码、状态码与异常处理
- 接口测试报告与缺陷记录

---

## 阅读路径

1. 先看基础认知，建立接口测试整体框架
2. 再看核心质量维度，明确接口到底该从哪些角度验证
3. 然后看工具与实践，理解这些能力怎么落地
4. 最后看进阶专题，把权限、异常表达和测试输出体系补齐

---

## 本专题重点

- 接口测试不是“把请求发出去”，而是验证接口契约、业务逻辑、数据结果和系统行为
- 对写操作接口，要始终同时关注返回值、数据库、缓存和链路结果
- 先建立“认知 -> 维度 -> 落地”的结构，再记工具和细节
