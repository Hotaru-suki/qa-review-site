# 结果验证与一致性

## 摘要

这一组知识点回答的是：接口不仅要能调用，还要把结果做对、把数据写对、把重复请求处理对。它聚焦“返回值正确”和“系统状态正确”之间的关系。

---

## 本组内容

- `可用性测试`
  先判断接口能不能进入正常处理。
- `正确性测试`
  再判断结果和业务规则是否正确。
- `缓存、数据库与读后写一致性`
  看接口返回成功后，用户立刻读取是否还能看到旧状态。
- `数据一致性测试`
  检查数据库、缓存和链路状态是否一致。
- `幂等性测试`
  检查重复请求下系统行为是否仍然正确。
- `消息去重、补偿与最终一致性`
  看异步链路里一条业务动作怎样被完整核对。

---

## 阅读重点

- 先判断接口能不能进入正常处理
- 再判断结果和业务规则是否正确
- 最后确认数据库、缓存和重复请求下的行为是否一致
