# 核心质量维度

## 摘要

这一组内容用来回答一个核心问题：接口除了“能调通”，还应该从哪些维度判断它是不是可靠。它不是简单列检查项，而是把不同质量风险拆成可分析、可验证的维度。

---

## 本组内容

- `可用性测试`
  看接口能不能进入正常处理。
- `正确性测试`
  看业务结果是不是对的。
- `健壮性测试`
  看异常输入和边界条件下是否可控。
- `稳定性测试`
  看长时间运行后是否退化。
- `接口性能测试`
  看负载上来后的响应和承载能力。
- `数据一致性测试`
  看数据库、缓存和链路结果是否一致。
- `幂等性测试`
  看重复请求下行为是否稳定正确。
- `兼容性测试`
  看不同版本或调用方式下是否稳定。
- `接口安全性测试`
  看越权、敏感信息和高风险操作是否被控制。

---

## 阅读重点

- 可用性和正确性负责判断接口有没有把事情做成
- 健壮性、安全性和兼容性负责判断风险场景是否可控
- 稳定性、性能、数据一致性和幂等性负责判断接口能不能支撑真实业务持续运行
