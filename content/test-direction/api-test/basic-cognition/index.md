# 接口测试中的基础认知

## 摘要

这一组内容是接口测试的起点，先把“接口测试是什么、通常怎么做、测试点是怎么拆出来的”这三件事讲清楚。它负责建立后面所有质量维度和工程实践的共同基础。

---

## 本组内容

- `接口测试概述`
  先明确接口测试在验证什么、边界在哪里。
- `接口测试流程`
  再看一次完整测试通常怎样推进。
- `接口测试用例设计思路`
  最后看测试点如何从输入、输出和状态变化里拆出来。
- `接口文档评审与测试点提炼`
  看拿到接口文档后，怎样快速识别风险和缺口。
- `前置条件、测试数据与依赖识别`
  看接口测试为什么经常不是只调一个 URL。

---

## 阅读重点

- 理解接口测试的边界
- 知道一次完整接口测试通常包含哪些环节
- 能从输入、输出、状态变化、异常处理四个维度设计用例
