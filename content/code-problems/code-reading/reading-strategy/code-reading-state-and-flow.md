# 如何追踪状态变化与调用链

## 摘要

读代码的真正难点通常不是看懂某一行，而是知道状态在什么时候被修改、沿着哪些调用链传播、最终怎样影响输出。

---

## 核心问题

- 哪些变量和对象值得重点跟踪
- 为什么状态变化往往比控制流本身更重要
- 怎样把调用链压缩成可理解的主路径

---

## 一个更稳的阅读顺序

阅读时应重点盯住：

- 输入参数
- 关键状态对象
- 写数据库、发请求、发消息等副作用点
- 异常返回和失败分支

这样才能知道一段代码到底在改变什么，而不只是“调用了很多函数”。

---

## 常见误区

- 所有函数都追一遍
- 只看调用，不看状态有没有变
- 只看正常路径，不看失败分支

---

## 小结

代码阅读的核心是跟状态，而不是跟所有行号。抓住状态变化，流程就容易看清。
