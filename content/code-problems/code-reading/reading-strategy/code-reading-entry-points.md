# 代码阅读从哪里开始

## 摘要

代码阅读最容易犯的错，是一开始就在细节里打转。更稳的做法是先找入口函数、核心对象和主流程，再逐步下钻。

---

## 核心问题

- 为什么入口选择会决定阅读效率
- 哪些位置最容易成为阅读起点
- 如何避免一开始就陷进工具函数和细枝末节

---

## 一个更稳的阅读顺序

代码阅读的常见入口包括：

- 请求入口或控制器
- 主流程函数
- 状态更新函数
- 异常和报错位置

从入口开始的价值在于先建立流程骨架，再补细节，而不是反过来。

---

## 常见误区

- 一上来先看工具函数
- 先追细节，不先看主流程
- 入口都没找到就开始下结论

---

## 小结

好的阅读起点会让代码先变成“流程图”，而不是一堆分散语句。
