# STL 选型与复杂度题怎么想

## 摘要

STL 相关题表面像“会不会 API”，实际上主要在考容器选择、访问模式和复杂度判断。

---

## 核心问题

- 题目为什么更关注你为什么选 `vector` / `map` / `priority_queue`
- 复杂度错误为什么会让看似正确的代码失分
- 怎样从操作模式反推出合适容器

---

## 一个更稳的思考顺序

这类题最重要的是先看：

- 按下标还是按 key 访问
- 要不要有序
- 是否需要快速取最大最小
- 插删发生在头尾还是中间

也就是说，先看“怎么操作数据”，再看“容器叫什么”。

---

## 常见判断方向

- 频繁尾插、随机访问：优先考虑 `vector`
- 需要按 key 快速查找：优先考虑 `unordered_map` 或 `map`
- 需要动态维护最值：优先考虑 `priority_queue`
- 需要稳定有序：优先考虑 `map` / `set` 这类有序结构

---

## 小结

STL 题的核心不是记接口，而是让容器和操作模式匹配。
