# 对象语义题怎么判断

## 摘要

C++ 对象语义题看起来像语法题，实际上在考对象生命周期、拷贝/移动、引用绑定和资源所有权。

---

## 核心问题

- 题目究竟在考值语义、引用语义还是资源管理
- 为什么很多输出题的本质是生命周期顺序题
- 面对构造、析构和赋值代码时应该先看什么

---

## 一个更稳的判断顺序

判断对象语义题时，最先看的是：

- 对象何时创建
- 对象是否被拷贝或移动
- 谁拥有资源
- 何时析构

这比盯着输出细节更重要。

---

## 常见误区

- 只盯最终输出，不看构造和析构顺序
- 分不清值传递和引用传递
- 忽略资源所有权，只把题当语法题

---

## 小结

对象语义题的本质，是把对象当成有生命周期和所有权的实体来看，而不是只看一行语法。
