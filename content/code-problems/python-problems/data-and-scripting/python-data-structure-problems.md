# Python 数据结构题怎么拆

## 摘要

Python 数据结构题通常不难写，但很容易因为类型选择和副作用理解不清而写出不稳代码。

---

## 核心问题

- 什么问题更适合 `list`、`dict`、`set`
- 可变对象为什么会影响题目结果
- 如何把遍历、过滤和统计写得清楚

---

## 一个更稳的解题顺序

先看问题更像：

- 顺序处理
- 键值映射
- 去重与成员判断

再决定数据结构，通常会比先写循环更稳。

---

## 常见判断方向

- 顺序遍历、保留顺序：优先想 `list`
- key 对应 value：优先想 `dict`
- 去重、快速判断是否存在：优先想 `set`

如果题里还涉及修改传入对象，就要额外留意副作用和可变对象问题。

---

## 小结

Python 数据结构题的关键不是短，而是选对类型、控制好副作用。
