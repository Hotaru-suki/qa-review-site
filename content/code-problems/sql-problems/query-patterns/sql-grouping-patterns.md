# 分组统计类 SQL 怎么拆

## 摘要

分组统计题常见但也最容易写错。难点通常不在 `GROUP BY` 本身，而在聚合条件、过滤顺序和结果口径。

---

## 核心问题

- 什么情况下必须先分组再过滤
- 聚合结果和原始行条件为什么不能混着写
- 统计题最常见的口径错误有哪些

---

## 一个更稳的解题顺序

这类题通常先看：

- 分组维度是什么
- 指标是什么
- 过滤发生在分组前还是分组后

看清这三点，SQL 骨架才会稳定。

---

## 常见误区

- 还没想清口径就先写 `GROUP BY`
- 把分组前过滤和分组后过滤混在一起
- 指标要的是“人数”，结果写成了“记录数”

---

## 小结

分组统计题的核心不是语法，而是口径。口径清楚，写法自然会顺。
